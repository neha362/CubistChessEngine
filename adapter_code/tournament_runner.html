Mix & Match — how it works
Every engine has three parts: move gen (legal moves), search (tree traversal strategy), and eval (position scoring).

In Mix & Match mode you independently choose the search and eval from different engine families. The move gen is always the shared base (all engines produce the same legal moves — the interesting variation is in search and eval).

Example: Siege search (aggressive α-β with quiescence) + Chaos eval (attack-proximity scoring) tests whether Siege's search discipline combined with Chaos's aggression metric outperforms either engine running pure.
build_combo() in Python
from engine_adapter import build_combo, run_tournament # build_combo(search, eval, max_depth, time_limit) combos = [ build_combo('classical_search', 'classical_eval'), # pure classical build_combo('siege_search', 'chaos_eval'), # siege search + chaos aggression build_combo('chaos_search', 'siege_eval'), # chaos search + siege king-zone build_combo('mcts_search', 'oracle_eval'), # MCTS + Claude scoring build_combo('classical_search', 'oracle_eval'), # deep search + Claude eval ] run_tournament(combos, max_moves=120)
Available search strategies
Key Description
'classical_search'Alpha-beta + iterative deepening + TT
'chaos_search'Negamax, check-first ordering, no quiescence
'siege_search'Alpha-beta + quiescence through checks
'mcts_search'UCB1 Monte Carlo tree, random rollouts
'nn_search'JS alpha-beta + MVV-LVA + history heuristic
Available eval functions
Key Description
'classical_eval'Material + piece-square tables + open file
'chaos_eval'Attack proximity, pawn storm, discounted material
'siege_eval'King-zone attacks, super-linear coordination bonus
'nn_eval'40 pattern rules (outposts, passed pawns, etc.)
'oracle_eval'Claude API scores position (slow, needs API key)
